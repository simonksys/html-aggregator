The 12 Principles of Animation

这些原则源自迪士尼经典著作《生命的幻象》(The Illusion of Life)，是赋予动态图形生命力的核心法则。 本课件通过 SVG 和 CSS 动画技术，为您直观演示每一条原则的物理逻辑与 Web UI 应用场景。

一、物理规律 (Physics)

01 挤压与伸展 (Squash & Stretch)

核心概念

物体不是刚体，在受力（加速或撞击）时会发生形变。必须保持**体积恒定**：压扁时必须变宽，拉长时必须变细。

UI 应用

点击按钮时，按钮微微缩小（挤压），松开时回弹；给用户一种“实体触感”而非冰冷的像素。

02 缓入缓出 (Slow In & Slow Out)

核心概念

自然界物体受惯性和摩擦力影响，不会瞬间达到最高速或瞬间静止。起步加速，停止减速。

UI 应用

几乎所有 UI 动画的标准：弹窗出现用 Ease-out，列表滚动用 Decay。纯线性运动 (Linear) 在界面中会显得非常廉价。

03 弧形运动 (Arcs)

核心概念

除了机械活塞，自然界生物的动作（如挥手、抛物）轨迹都是弧线。

UI 应用

“加入购物车”的动效，商品图标飞入购物车的路径如果是直线的会很生硬，带一点弧度会更自然流畅。

04 跟随与重叠 (Follow Through)

核心概念

主体停下后，附属物（如尾巴、头发）因惯性继续运动。动作不是同时发生，而是有相位差。

UI 应用

下拉刷新停止后，图标的惯性摆动；或者加载卡片列表时，卡片不是同时出现，而是依次错开 (Stagger) 出现。

二、动作表演 (Acting)

05 预备动作 (Anticipation)

核心概念

在主动作发生前，向反方向做一个小动作。目的是积蓄能量，并**提示观众**“我要动了”。

UI 应用

删除项目前，项目微微发红或抖动；悬停按钮时按钮轻微后缩，预示可点击。

06 舞台表现 (Staging)

核心概念

引导观众视线。通过对比（明暗、动静、大小）突出主体，一次只表达一个重点。

UI 应用

模态弹窗 (Modal) 出现时背景变黑（遮罩），强迫用户聚焦弹窗内容，这就是 Staging。

07 时间节奏 (Timing)

核心概念

动作持续的时间。时间决定了**重量**（快=轻，慢=重）和**情绪**（慢=悲伤/谨慎，快=活力）。

UI 应用

报错提示应快速出现（警示），而成功提示可以稍慢（愉悦）。不同操作反馈的快慢直接影响产品的“手感”。

08 次要动作 (Secondary Action)

核心概念

丰富主动作的细节（如走路时摆手）。次要动作不能抢戏，只能衬托。

UI 应用

上传文件时，进度条增长是主动作，进度条上的光效闪烁是次要动作，暗示系统正在忙碌。

三、美术设计 (Design)

09 逐帧与关键帧

核心概念

Pose to Pose（关键帧）结构严谨，由电脑补间（大多数 Web 动画）。Straight Ahead（逐帧）狂野、随机，具有手绘质感。

UI 应用

常规交互使用关键帧（CSS Transition）。点赞爆炸、礼花特效通常使用逐帧图（Sprite Sheet）或 Lottie 来表现那种不可预测的动态。

10 夸张 (Exaggeration)

核心概念

基于现实但超越现实。通过大幅度的变形、震动来强化情绪和冲击力。

UI 应用

输入密码错误时，输入框左右摇晃的幅度通常比物理摇晃要剧烈，目的是明确地告知用户“出错了”。

11 扎实的描绘 (Solid Drawing)

核心概念

在 2D 平面表现 3D 体积感。避免“纸片人”。利用透视、光影和视差 (Parallax)。

UI 应用

Material Design 的阴影层级（Elevation）；或者像演示中那样，利用面部五官移动距离的不同（视差）来模拟头部转动。

12 吸引力 (Appeal)

核心概念

好的设计让人赏心悦目。通过形状（圆形优于方形）、色彩和动态赋予角色或界面“魅力”。

UI 应用

Micro-interactions（微交互）。比如 Twitter 点赞时的红心动画，或者加载时的可爱动画。好的 UX 应该让人感到愉悦。
